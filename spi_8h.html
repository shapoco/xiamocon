◆ xmc_spi_dma_write_start()

xmc_status_t xmc_spi_dma_write_start ( const xmc_dma_config_t *  cfg,
int  cs_pin 
)

Start a DMA-based SPI write operation. This will return immediately after starting the DMA transfer, and the caller must call xmc_spi_dma_complete() to wait for the transfer to finish.

Parameters
cfgPointer to the DMA configuration for the transfer.
cs_pinThe GPIO pin number to use for the chip select (CS) line during the transfer, or -1 to not control any CS line. If a valid pin number is provided, the CS line will be pulled low at the start of the transfer and pulled high when the transfer is complete.
Returns
XMC_OK if the DMA transfer was successfully started, or an error code if there was a problem with the configuration.
